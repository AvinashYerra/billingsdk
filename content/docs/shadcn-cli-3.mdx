### Namespaced Registry
With shadCN CLI 3.0, you can use namespaced registry to add components to your project easily.

Edit `components.json` with following code to add billingSDK components to your project:

```json title="components.json"
{
  "registries": {
    "@billingsdk": "https://billingsdk.com/r/{name}.json"
  }
}
```

Now you can add components to your project with the following command:

<Tabs items={['npx', 'pnpm', 'yarn', 'bun']} defaultValue="npx" groupId="shadcn-cli-3-namespaced-registry-tabs">
    <Tab value="npx">
    ```bash
    npx shadcn@latest add @billingsdk/[component-name]
    ```
    </Tab>
    <Tab value="pnpm">
    ```bash
    pnpm dlx shadcn@latest add @billingsdk/[component-name]
    ```
    </Tab>
    <Tab value="yarn">
    ```bash
    npx shadcn@latest add @billingsdk/[component-name]
    ```
    </Tab>
    <Tab value="bun">
    ```bash
    npx shadcn@latest add @billingsdk/[component-name]
    ```
    </Tab>
</Tabs>

Example:

```bash
npx shadcn@latest add @billingsdk/pricing-table-one
```

## MCP Support
With shadcn cli 3.0, you can use MCP to add components to your project.
With this, your AI assistant can help you add components to your project with just a few words.

Add **`@billingsdk`** to your `components.json`:

```json title="components.json"
{
  "registries": {
    "@billingsdk": "https://billingsdk.com/r/{name}.json"
  }
}
```

Add the MCP server to your project:

<Tabs items={['npx', 'pnpm', 'yarn', 'bun']} defaultValue="npx" groupId="shadcn-cli-3-mcp-server-tabs">
    <Tab value="npx">
    ```bash
    npx shadcn@latest mcp init
    ```
    </Tab>
    <Tab value="pnpm">
    ```bash
    pnpm dlx shadcn@latest mcp init
    ```
    </Tab>
    <Tab value="yarn">
    ```bash
    npx shadcn@latest mcp init
    ```
    </Tab>
    <Tab value="bun">
    ```bash
    npx shadcn@latest mcp init
    ```
    </Tab>
</Tabs>

That's it! Now you can ask AI with a prompt like this:

```
Add a pricing table to my project
```

or

```
Add a banner to my project
```
