## Usage

```tsx
import { PaymentMethodManagerUPI, type PaymentMethodUPI } from "@/registry/billingsdk/payment-method-manager-upi";
```

```tsx
const [upiMethods, setUpiMethods] = useState<PaymentMethodUPI[]>([
  { id: "upi1", vpa: "user@upi", bank: "HDFC", isDefault: true },
  { id: "upi2", vpa: "another@upi", bank: "ICICI", isDefault: false },
]);
```

```tsx
<PaymentMethodManagerUPI
  className="max-w-4xl mx-auto"
  paymentMethods={upiMethods}
  onAdd={(method) => setUpiMethods([...upiMethods, method])}
  onEdit={(updated) => setUpiMethods(upiMethods.map(m => m.id === updated.id ? updated : m))}
  onRemove={(id) => setUpiMethods(upiMethods.filter(m => m.id !== id))}
  onSetDefault={(id) => setUpiMethods(upiMethods.map(m => ({ ...m, isDefault: m.id === id })))}
/>
```

## Props

| Prop             | Type                                   | Required | Description                                 |
| ---------------- | -------------------------------------- | -------- | ------------------------------------------- |
| `className`      | `string`                               | ❌        | Additional CSS classes for styling          |
| `paymentMethods` | `PaymentMethodUPI[]`                   | ✅        | Array of UPI payment methods to display     |
| `onAdd`          | `(method: PaymentMethodUPI) => void`   | ✅        | Callback when a new UPI method is added     |
| `onEdit`         | `(method: PaymentMethodUPI) => void`   | ✅        | Callback when a UPI method is edited        |
| `onRemove`       | `(id: string) => void`                 | ✅        | Callback when a UPI method is removed       |
| `onSetDefault`   | `(id: string) => void`                 | ✅        | Callback to set a UPI method as default     |

## PaymentMethodUPI Interface

```tsx
interface PaymentMethodUPI {
  id: string;
  vpa: string;
  bank?: string;
  isDefault: boolean;
}
```

## Theming

The Payment Method Manager UPI component is styled using the `shadcn/ui` library. You can customize the colors and fonts by overriding the CSS variables. You can also get the theme from the [Theming](/docs/theming) page.
